JavaScript Text Editor

A browser-based rich text editor built with Express.js and Handlebars, featuring live formatting controls, theme switching, and real-time counters.

GitHub

Objective

Build a fully interactive, browser-based text editor powered by an Express.js server and Handlebars templating. The editor needed to support a wide range of formatting tools: font family and size controls, bold/italic/underline, text and background color via RGB sliders, text alignment, find-and-replace, image insertion, margin and padding adjustment, and multiple selectable themes. All of these features are driven by DOM manipulation and event handling in vanilla JavaScript.

Development Process

💡 Planning & Ideation

Before writing any code, I studied the provided CodePen reference to understand how the editor should look and behave, paying attention to toolbar groupings, button layout, and edge cases like multiple consecutive spaces. I then planned the project file structure: an Express app with assets in public/ and Handlebars templates in views/, mirroring the assignment requirements.

💡 Implementation

I started with app.js, configuring Express, routing, and path handling, then moved to the Handlebars template (index.hbs). The toolbar was organized into semantic grouping divs, keeping related controls (e.g. bold, italic, underline) together and separated by visual dividers. I brought in Google Fonts for five typeface options and a font icon library for toolbar icons.

The JavaScript file handled all interactivity. I grabbed element references via getElementById, then built event listeners for each toolbar control in order of complexity, starting from basic text formatting up to image insertion and live counters. Functions were modularized to avoid code duplication, especially for the shared RGB color palette used by both text and background color pickers.

💡 Challenges & Solutions

DOM Formatting: Applying styles only to selected text, rather than the whole editor, required learning the document.execCommand() API, which wraps selections in the appropriate HTML elements. This unlocked most of the text formatting features.

Event Types: Several controls needed input event listeners rather than click (for instance, the RGB sliders and font size field), a distinction I worked through by testing and debugging iteratively.

Live Counters: Keeping character, word, and line counts updated in real time was tricky. I implemented a single updateCounters() function that runs on every input event, calculating counts by splitting the editor content appropriately and updating the display elements.

Server Workflow: Without a live-reload setup, every CSS or server-side change required manually restarting the Node server, which slowed styling iteration significantly. I adapted by batching visual changes before restarting.

Project Impact

The finished editor handles all required formatting features with a clean, themed UI. The project pushed my understanding of DOM manipulation, modular JavaScript architecture, and full-stack Node.js app structure well beyond what previous projects had covered.

  • Mastered DOM selection and execCommand-based rich text formatting.
  • Built a modular, reusable RGB color palette component shared across two features.
  • Implemented real-time counters tracking characters, words, lines, and elapsed edit time.
  • Gained hands-on experience with Express.js routing and Handlebars templating.
  • Improved problem-solving skills through iterative debugging and testing.
  • Strengthened ability to collaborate with AI-assisted development tools.

Development Reflection

Looking back, the project grew from a blank Express app to a fully featured in-browser text editor with a polished, themed interface. The JavaScript side was the steepest learning curve, as I had to wrestle with event types, selection-aware formatting, and debugging a live DOM without the convenience of a hot-reload setup. CSS came most naturally, though the constant server restarts made it slower than expected. Overall, this project was the most technically comprehensive one I've built, and it gave me a much clearer picture of how front-end interactivity, server-side routing, and templating fit together in a real application.
