Object Oriented Programming: Design Patterns Project

A Java-based customizable e-commerce system designed using core object-oriented design patterns to support flexible product customization, payments, and notifications.

GitHub Project Report (PDF)

Objective

The objective of this project was to design and implement a scalable, customizable online store in Java that supports dynamic product customization, multiple payment methods, and real-time user notifications while remaining easy to maintain and extend.

Development Process

💡 Planning & Ideation

The system was built around object-oriented design patterns to ensure modularity, extensibility, and clean separation of responsibilities. Each team member was responsible for implementing specific design patterns, requiring careful planning to ensure compatibility and consistency across the codebase.
Before implementation, we identified key functional requirements such as customizable products, flexible payments, and user notifications—and mapped each requirement to an appropriate design pattern.

💡 Implementation

The application was implemented in Java and integrates five core design patterns to manage complexity and promote scalability:

  • Factory Pattern: Used to create different types of products (e.g., electronics, clothing) based on user selections, allowing easy addition of new product categories.
  • Decorator Pattern: Enabled dynamic addition of features to products (e.g., color, text, size) without modifying existing classes.
  • Strategy Pattern: Allowed users to choose from multiple payment methods (e.g., credit card, PayPal) at checkout, encapsulating each payment algorithm.
  • Observer Pattern: Implemented a notification system that alerts users about order status updates.
  • Model-View-Controller (MVC): Separated the application into three interconnected components to manage application logic, user interface, and user interactions.

💡 Challenges & Solutions

Technical Challenge: Managing a growing number of product customization combinations without making the codebase overly complex.

The Solution: My group used the Decorator pattern to allow customization options to be implemented as independent, stackable components. This kept the system modular and avoided the need for a large number of specialized subclasses.

Project Impact

The final system successfully demonstrates how object-oriented design patterns can be combined to build a flexible and maintainable application.

  • Implemented a fully customizable product system without redundant classes
  • Enabled dynamic payment and notification behaviors through interchangeable components
  • Built a clean, scalable Java architecture suitable for future expansion
  • Strengthened skills in practical application of design patterns

Development Reflection

This project strengthened my understanding of how thoughtful architecture directly impacts scalability and maintainability. By applying multiple design patterns within a single system, I learned how to balance flexibility with clarity while working within a shared codebase.
The experience also reinforced the importance of communication and consistency when collaborating on complex software systems which translate directly to real-world development environments.
