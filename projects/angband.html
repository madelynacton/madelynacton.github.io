Angband Monster File Editor

A Python command-line tool to search, view, and edit monster data for the game Angband, with robust input validation and file management.

GitHub

Objective

The goal was to create a user-friendly command-line interface for editing monster attributes in Angband's "monster.txt" file. The program needed to allow safe editing, search functionality, and backup handling to prevent data loss.

Development Process

💡 Planning & Ideation

My team and I analyzed the structure of Angband's monster data and designed a command-line workflow with search, list, and edit functionalities. Initially, we planned multiple screens to simulate a GUI-like interface but later simplified to a single interactive loop for better usability.

💡 Implementation

I developed the Python backend to load, edit, and save monster data with input validation for attributes like HP, speed, flags, and blows. The program also creates a backup file ("monster_orig.txt") and only writes changes to "monster.txt" upon exit. Other team members worked on graph visualizations, logs, and documentation.

💡 Challenges & Solutions

Technical Challenge: Handling diverse user input and ensuring changes were safely written back without corrupting the original data.

The Solution: Created a mapping system for attribute names, robust input validation, and a static attribute list in the Monster class. All changes are written only on program exit to preserve data integrity.

Project Impact

The final MFE allows users to search, list, and edit monsters with confidence, improving Angband gameplay customization. The program ensures backups, input validation, and proper data structure handling.

  • Developed a reliable Python CLI tool for managing Angband monsters.
  • Gained experience in backend logic, file I/O, data validation, and team-based software development.
  • Implemented a robust search and edit system with attribute mapping for intuitive user input.
  • Designed a safe backup workflow to prevent data loss while allowing persistent edits.
  • Improved understanding of handling complex data structures in Python, including lists and classes.
  • Enhanced problem-solving skills by addressing edge cases and unexpected user input scenarios.
  • Collaborated effectively with teammates on testing, documentation, and additional visualization features.

Development Reflection

Throughout the span of approximate three weeks, the project grew from a complex, multi-screen CLI design to a streamlined, single-loop interface that is easier to use and maintain. Simplifying the design improved usability, testing efficiency, and overall team collaboration.
