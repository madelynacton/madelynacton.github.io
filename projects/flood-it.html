Flood It! Game

A Java implementation of the Flood It puzzle game focused on graph traversal, algorithm optimization, and runtime analysis.

GitHub

Objective

The goal of this project was to implement a working version of the Flood It puzzle game while applying core data structures and algorithms concepts. The project emphasized flood-fill logic, coordinate-based grid traversal, and analyzing the performance of different algorithmic approaches.

Development Process

💡 Planning & Ideation

The project was planned by outlining core features such as flood-fill algorithms, board representation, and performance measurement. We designed the data structures to track flooded regions and identified how to test algorithm efficiency across different board sizes.

💡 Implementation

The game was implemented in Java using a model-view-controller structure. The board is a 2D array of tiles, with a linked list tracking flooded regions and queues for breadth-first traversal. Two flood-fill algorithms were implemented: iterative expansion and BFS. The GUI was built with Swing, and automated batch testing was used for performance evaluation.

💡 Challenges & Solutions

Technical Challenge: Efficiently tracking flooded regions while preventing revisiting tiles and ensuring BFS/iterative algorithms produce correct results.

The Solution: Used LinkedList and Queue structures for BFS, HashSet for visited tiles, and careful iteration over neighbors. Automated tests verified correctness and performance scaling.

Project Impact

The project successfully reproduces Flood It gameplay and demonstrates algorithm efficiency differences in iterative vs. BFS flood-fill approaches. It strengthened skills in Java, data structures, algorithm optimization, and performance analysis.

  • Implemented flood-fill using BFS and iterative expansion for performance comparison.
  • Used LinkedList and Queue structures to track flooded tiles efficiently.
  • Applied HashSet tracking to prevent revisiting tiles and optimize traversal.
  • Modeled the board as a 2D array, treating the grid as an implicit graph.
  • Analyzed runtime performance across increasing board sizes with automated tests.
  • Strengthened skills in debugging, optimizing, and extending a multi-file Java codebase.

Development Reflection

This project evolved from a simple puzzle game idea into a fully-featured implementation emphasizing algorithm design, data structures, and performance analysis. Iterative testing and GUI development reinforced Java coding practices and problem-solving skills.
